# VibeFly

**Phone-as-a-Server.** Превращает Android-телефон в полноценный PaaS с AI-ассистентом, который умеет деплоить и поддерживать сайты, боты и API.

Один APK. Внутри — настоящий Linux с systemd, контейнерами, нашим управляющим агентом и AI, который понимает что у тебя на сервере и помогает с этим жить.

> Старый Pixel в ящике стола → твой личный Coolify за 10 минут.

---

## Идея

Современные смартфоны мощнее, чем дешёвые VPS. У бюджетного Android 2024 года 6–8 ГБ RAM, UFS-накопитель быстрее SATA-SSD, ARM-чип сильнее Raspberry Pi 4. Электричества жрёт в десять раз меньше десктопа, имеет встроенный UPS (батарею), бесшумный, помещается в карман.

Проблема одна — это всё намертво заперто внутри Android. Никакого systemd, никакого Docker, никакого нормального хостинга. Существующие решения (Termux, Andronix, KSWEB) либо для разработчиков с CLI, либо игрушки для статики.

VibeFly закрывает разрыв.

## Что внутри APK

```
┌─────────────────────────────────────────┐
│  UI (Kotlin + Jetpack Compose)          │
│  Дашборд, AI-чат, marketplace, логи     │
├─────────────────────────────────────────┤
│  Agent (Go, статический бинарник)       │
│  REST + WebSocket, tools для AI,        │
│  оркестрация контейнеров, метрики       │
├─────────────────────────────────────────┤
│  Runtime (Droidspaces / namespaces)     │
│  systemd как PID 1, cgroups, изоляция   │
├─────────────────────────────────────────┤
│  Rootfs.img (Debian 12 ARM64)           │
│  node, python, postgres, redis, nginx,  │
│  pm2, marketplace-приложения            │
└─────────────────────────────────────────┘
```

Пользователь видит **одно приложение**. Установил, открыл, тапнул Start — у него есть сервер с реальным Linux, доменом через Cloudflare Tunnel и AI-ассистентом.

## Что умеет VibeFly

**Хостинг.** Деплой любого Node / Python / Go / Rust / PHP приложения из git. Автоматический build, доменное имя, HTTPS, реверс-прокси, лог-агрегация.

**Управление.** Старт, стоп, рестарт, environment variables, история деплоев, бэкапы, rollback на предыдущую версию одной кнопкой.

**Мониторинг.** Метрики устройства (CPU, RAM, температура, батарея, троттлинг). Метрики приложений. Алерты на падения, перегрев, OOM.

**AI-ассистент.** Чат, который видит твой сервер. Объясняет логи, находит причины падений, предлагает миграции БД, генерирует код для новых приложений. Все действия — через подтверждение пользователя, никаких сюрпризов.

**Marketplace.** One-click install для популярных self-hosted приложений: n8n, Vaultwarden, Uptime Kuma, PostgreSQL+pgAdmin, AdGuard, Memos, Pi-hole, и так далее.

**Сеть.** Cloudflare Tunnel из коробки — публичный домен без проброса портов. Tailscale — приватная сеть для управления. Можно и без всего этого — локально по Wi-Fi.

## Почему это сейчас имеет смысл

- Самохостинг растёт. r/selfhosted перешагнул 700k, eco-движение в ЕС толкает «repair, not replace».
- AI снимает порог входа. Раньше для запуска приложения нужен был DevOps. Теперь AI делает за пользователя 80%, оставшиеся 20% — подтверждения.
- Старых телефонов у людей десятки. Они лежат в ящиках, потому что Android считает их «устаревшими». На самом деле — это серверы, ждущие своей второй жизни.
- Coolify и аналоги показали, что PaaS-уровня контроля хочет уже не только корпорация. Но Coolify требует VPS, Docker и навыков. VibeFly — один тап.

## Не цели

- Заменить AWS / Vercel / Cloudflare для прод-нагрузок. Это для домашнего и небольшого коммерческого хостинга.
- Замена смартфону. Тестовый телефон должен жить отдельно от ежедневного.
- Серверы, требующие настоящего железа: ML-тренировки, GPU-инференс крупных моделей, тяжёлые БД на терабайты.

## Состояние проекта

**Фаза 0 — Proof of Concept.** Проверяем что embedded Linux реально работает на современных Android. Тестовое устройство — Redmi Note 14S с root.

Подробный план — в [ROADMAP.md](./ROADMAP.md).

## Стек

- **APK:** Kotlin, Jetpack Compose, Coroutines, Foreground Service
- **Runtime:** Droidspaces (Linux namespaces, статический C/musl бинарник)
- **Rootfs:** Debian 12 ARM64, systemd, debootstrap
- **Agent:** Go, статическая сборка, без зависимостей
- **Cloud (опционально):** Next.js 15, Prisma, PostgreSQL, NextAuth
- **AI:** OpenRouter / Anthropic / OpenAI / локальные модели через Cloudflare Worker proxy
