# UI-мокапы

На стадии прототипирования сделаны мокапы в двух стилевых направлениях:

## 1. Современный mobile (production-кандидат)

Чистый mobile-first дизайн с акцентом на функциональность. Дашборд с метриками устройства, список приложений, чат с AI с tool-call card'ами и approval-кнопками, marketplace one-click apps.

Характерные элементы:

- Минималистичные карточки приложений со статусом
- Health-индикатор устройства (battery / temp / cpu / ram) первым на экране
- Bottom navigation: apps / metrics / logs / settings
- Approval card'ы для опасных AI-действий с preview SQL / diff

## 2. Skeuomorphic iOS 6 (стилистический эксперимент)

Полностью аутентичный язык эпохи Forstall: глянцевые иконки с inset-highlight'ом, linen-текстура фона, выпуклые кнопки, brushed-metal tab bar, кожаная подложка для логов с фосфорно-зелёным CRT-стилем терминала.

Экраны: Springboard в стиле iPhone 4S, Telegram-style чат с AI с avatara в фиолетовом стекле, Settings.app-layout настроек, app detail с grouped table view.

Художественная ценность высокая, но для массового продукта 2026 года выберем направление 1.

## Где они хранятся

Мокапы рендерились через визуализатор Claude как inline-виджеты (HTML+CSS). Полноценные Figma-файлы будут созданы ближе к фазе 2 при старте Android-проекта.

Ключевые скриншоты будут сохранены в `docs/screenshots/` при стабилизации UI-концепта.

## Принципы дизайна

1. **Mobile-first без компромиссов.** UI оптимизирован для одной руки, никаких desktop-портов.
2. **Прозрачность AI-действий.** Tool calls всегда видны, опасные действия требуют подтверждения.
3. **Health на первом экране.** Состояние устройства (battery, temp) важнее списка приложений.
4. **Минимум формы.** Дефолты выбраны так, чтобы deploy был возможен в 2 тапа.
